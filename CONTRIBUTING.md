贡献指南
感谢你有兴趣为这个局域网文件互传工具做贡献！即使是零基础的新手，只要愿意参与，都能帮上忙。这份指南会告诉你如何把项目跑起来，以及如何提交你的修改。
快速开始：把项目跑起来
你的电脑上需要先装好 Python（建议 3.8 及以上版本）。然后按下面几步操作：

1. 下载项目代码
   - 先点击仓库右上角的 “Fork” 按钮，把项目复制到你自己的 GitHub 账号下。
   - 然后把你 Fork 后的仓库克隆到本地：
     - 打开终端（或命令提示符），进入你想存放项目的文件夹，运行你仓库的克隆地址（地址在你仓库的 “Code” 按钮下可以复制）。

2. 安装项目依赖
   - 在终端中进入项目文件夹，执行：
     - `pip install -r requirements.txt`
   - 如果没有 `requirements.txt` 文件，就手动安装需要的包：
     - `pip install flask flask-socketio flask-cors qrcode[pil] Pillow`

3. 启动后端服务
   - 在项目文件夹下执行：
     - `python app.py`
   - 成功后终端会显示一个局域网地址和一个6位数字的会话码，表示服务已启动。

4. 访问前端页面
   - 在电脑浏览器中打开终端显示的局域网地址（例如 `http://192.168.1.5:8080`），就可以看到加入页面了。
   - 用其他设备（手机、平板）连到同一个 WiFi，扫描页面上的二维码，或手动输入会话码，就能进入同一个文件共享对话。

如何贡献代码

我们使用 **Fork + Pull Request** 的流程，步骤很简单：

1. 创建你的修改分支
   - 在你 Fork 后的仓库里，创建一个新分支，分支名最好能说明你要做什么，比如 `feature/add-dark-mode` 或 `fix/pdf-convert-error`。

2. 做出你的修改
   - 在本地编写代码，然后测试一下，确保修改没有破坏原有的功能。
   - 提交代码时，写一条清晰的提交说明（例如 “修复 PDF 转换时 LibreOffice 路径错误”）。

3. 提交 Pull Request
   - 当你对自己的改动满意后，把你仓库里的新分支推送到 GitHub。
   - 回到原始项目仓库，你会看到 “Compare & pull request” 的按钮，点击它。
   - 在 Pull Request 的描述中，说说你做了什么修改，为什么要这样改，如果有相关的 Issue 也可以链接上。
   - 提交后，项目维护者会进行审核，可能会和你讨论一些细节。

代码风格
- 变量名、函数名尽量清晰，用英文小写加下划线命名，比如 `get_local_ip`。
- 中文注释只用在需要向用户解释的地方，其他代码注释建议用英文或简中均可，保持统一。
- 保持和现有代码类似的缩进和排版，不要引入大型的格式化工具。

提交 Issue
如果你发现 bug，或者有新的功能想法，可以在 Issues 区提交。写清楚你的使用场景、步骤和遇到的问题，如果能附上截图或复现步骤就更好了。我们会尽快处理。
再次感谢你的参与！哪怕是一个标点符号的修正，或是一个新的界面优化点子，都是对这个项目莫大的帮助。
